# Scratch Startup Checklist

Use this checklist at the start of work in any `scratch` subfolder.

1. Confirm the current working directory and project target.
2. Read `PROJECT_MEMORY.md`.
3. Read `AGENTS.md`.
4. Check whether the workspace is already a git repository.
5. If git is missing and the task involves meaningful edits, initialize git first.
6. Note any project split or deployment mode differences before changing shared files.
7. After meaningful edits, create a task-scoped git commit.
