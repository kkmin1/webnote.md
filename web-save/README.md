# MHTML Web Saver

로컬에서 실행하는 작은 웹 저장 도구입니다. 브라우저에서 URL을 입력하면 FastAPI 백엔드가 페이지와 리소스를 모아 단일 `.mhtml` 파일로 내려줍니다.

## 폴더 분석

- `server.py`: FastAPI 앱. `/save`, `/proxy`, `/assemble`, `/health`, `/` 엔드포인트를 제공합니다.
- `index.html`: 별도 빌드 없이 바로 서빙되는 단일 파일 UI입니다.
- `requirements.txt`: 백엔드 실행에 필요한 최소 의존성 목록입니다.

## 동작 방식

1. 사용자가 URL을 입력하면 프론트엔드가 `POST /save`를 먼저 호출합니다.
2. 서버가 직접 HTML과 리소스를 수집해 MHTML을 만들 수 있으면 바로 다운로드합니다.
3. 직접 수집이 실패하면 브라우저가 페이지 HTML을 가져오고, 서버가 `/proxy`로 리소스를 대신 수집합니다.
4. 마지막에 `/assemble`이 HTML과 수집된 리소스를 하나의 MHTML로 조립합니다.

이 구조 덕분에 일반 페이지와 일부 인증/차단 환경을 모두 어느 정도 대응할 수 있습니다.

## 실행 방법

```bash
pip install -r requirements.txt
uvicorn server:app --reload --port 8765
```

브라우저에서 [http://localhost:8765](http://localhost:8765) 로 접속합니다.

## 사용 방법

1. 저장할 웹페이지 URL을 입력합니다.
2. `Enter` 또는 `저장` 버튼을 눌러 수집을 시작합니다.
3. 완료되면 `.mhtml` 파일이 자동 다운로드됩니다.
4. 하단 저장 이력에서 최근 성공/실패 기록과 재시도를 확인할 수 있습니다.

## 현재 장점

- 프론트엔드와 백엔드가 모두 단순해서 수정 범위가 작습니다.
- `/save` 실패 시 폴백 경로가 있어 단순 fetch 실패에 덜 민감합니다.
- 저장 이력, 서버 상태 확인, 재시도 버튼이 이미 포함돼 있습니다.

## 한계

- JavaScript 실행 이후에 동적으로 만들어지는 화면은 완전하게 보존되지 않을 수 있습니다.
- 브라우저 쿠키를 실제로 넘기지 않으므로 로그인 기반 페이지 저장은 제한적입니다.
- 외부 폰트는 CDN 의존이 있으므로 오프라인 상태에서는 UI 글꼴이 대체 폰트로 보일 수 있습니다.
